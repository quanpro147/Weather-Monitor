==============================================================
I. TỔNG QUAN HỆ THỐNG
==============================================================

Weather Monitor là nền tảng giám sát thời tiết và chất lượng
không khí cho các tỉnh thành Việt Nam. Hệ thống thu thập dữ
liệu từ API mở (Open-Meteo), xử lý, lưu trữ, phân tích bằng
ML và hiển thị trên dashboard web.

Công nghệ chính:
  - Frontend  : Next.js 14 + TypeScript + Tailwind CSS
  - Backend   : FastAPI (Python)
  - Database  : Supabase (PostgreSQL cloud)
  - Cache     : Redis
  - ML        : scikit-learn (Isolation Forest)
  - Data API  : Open-Meteo Archive API (miễn phí, không cần key)
  - LLM       : Gemini / OpenAI (tóm tắt AI — chưa triển khai)
  - Container : Docker Compose

--------------------------------------------------------------
3.1. DATA PIPELINE — data-pipeline/fetchers/Collect_RealTime.py
--------------------------------------------------------------
Chức năng:
  - Nguồn dữ liệu: Open-Meteo Archive API
    URL: https://archive-api.open-meteo.com/v1/archive
  - Tham số: latitude, longitude, start_date, end_date, daily
  - Biến thu thập: temperature_2m_max/min/mean, rain_sum,
    wind_speed_10m_max, relative_humidity_2m_mean, cloud_cover_mean

Luồng xử lý:
  1. Đọc VietNam.csv → danh sách tỉnh thành (tên, lat, lon)
  2. Upsert bảng cities (psycopg2, ON CONFLICT DO NOTHING)
  3. Với mỗi thành phố:
     a. Query ngày mới nhất đã có → start = ngày đó + 1
     b. Nếu chưa có dữ liệu → start = 2024-01-01
     c. Gọi Open-Meteo, nhận JSON
     d. Xử lý 429 rate limit:
        - minutely_usage > 9000 → sleep 60s
        - hourly_usage > 9000   → sleep 3600s
     e. Parse daily arrays → list of records
     f. Executemany INSERT INTO weather_daily
  4. Lên lịch chạy mỗi ngày lúc 02:00 (schedule lib)

Lệnh chạy:
  python data-pipeline/fetchers/Collect_RealTime.py

--------------------------------------------------------------
3.2. DATABASE SCHEMA — infra/supabase/schema.sql
--------------------------------------------------------------
Bảng cities:
  id         SERIAL PRIMARY KEY
  name       VARCHAR(100)
  latitude   DECIMAL(9,6)
  longitude  DECIMAL(9,6)
  country    VARCHAR(100) DEFAULT 'Vietnam'
  timezone   VARCHAR(50) DEFAULT 'Asia/Ho_Chi_Minh'
  created_at TIMESTAMP DEFAULT NOW()

Bảng weather_daily:
  id                          SERIAL PRIMARY KEY
  city_id                     INT REFERENCES cities(id)
  date                        DATE
  temperature_2m_max          DECIMAL(5,2)
  temperature_2m_min          DECIMAL(5,2)
  temperature_2m_mean         DECIMAL(5,2)
  rain_sum                    DECIMAL(8,2)
  wind_speed_10m_max          DECIMAL(6,2)
  relative_humidity_2m_mean   DECIMAL(5,2)
  cloud_cover_mean            DECIMAL(5,2)
  created_at                  TIMESTAMP DEFAULT NOW()
  UNIQUE(city_id, date)

--------------------------------------------------------------
3.3. REDIS CACHE — infra/redis/redis.conf
--------------------------------------------------------------
Mục đích: Giảm tải cho Supabase, giảm latency API

Cache keys và TTL:
  cities_list              → 86400s (24 giờ) — ít thay đổi
  weather:{city_id}:{start}:{end}  → 3600s  (1 giờ)
  anomaly:{city_id}:{start}:{end}  → 21600s (6 giờ)

Cài đặt:
  REDIS_HOST=localhost (hoặc "redis" trong Docker)
  REDIS_PORT=6379
  REDIS_DB=0

--------------------------------------------------------------
3.4. FASTAPI MICROSERVICE — services/api/
--------------------------------------------------------------
Cấu trúc:
  services/api/
  ├── main.py              # App entry, CORS, register routers
  ├── app/
  │   ├── routers/
  │   │   ├── cities.py    # GET /cities/
  │   │   ├── weather.py   # GET /weather/{city_id}  + advisory
  │   │   └── anomaly.py   # GET /anomaly/{city_id}
  │   ├── models/
  │   │   ├── weather.py   # Pydantic WeatherRecord
  │   │   └── anomaly.py   # Pydantic AnomalyRecord
  │   └── ml/
  │       └── anomaly.py   # Isolation Forest logic

Endpoints:
  GET /cities/
    - Lấy danh sách thành phố từ Supabase
    - Cache: cities_list, TTL 24h

  GET /weather/{city_id}?start_date=&end_date=
    - Lấy dữ liệu thời tiết theo khoảng ngày
    - Cache: weather:{city_id}:{start}:{end}, TTL 1h
    - Kèm advisory (cảnh báo quy tắc cố định)

  GET /anomaly/{city_id}?start_date=&end_date=
    - Fetch TOÀN BỘ lịch sử thành phố để huấn luyện mô hình
    - Chạy ML inference (IsolationForest)
    - Lọc kết quả theo khoảng ngày yêu cầu
    - Cache: anomaly:{city_id}:{start}:{end}, TTL 6h

ML Advisory (weather.py — rule-based):
  Ngưỡng cảnh báo hiện tại (hardcoded):
  - Nhiệt độ max > 38°C  → nắng nóng cực đoan
  - Lượng mưa > 50mm     → nguy cơ lũ lụt
  - Tốc độ gió > 60km/h  → gió mạnh nguy hiểm

--------------------------------------------------------------
3.5. ML — ANOMALY DETECTION (services/api/app/ml/anomaly.py)
--------------------------------------------------------------
Thuật toán: Isolation Forest (unsupervised)
Thư viện: scikit-learn

Cấu hình:
  FEATURES = [
    "temperature_2m_max", "temperature_2m_min",
    "temperature_2m_mean", "rain_sum",
    "wind_speed_10m_max", "relative_humidity_2m_mean",
    "cloud_cover_mean"
  ]
  CONTAMINATION = 0.05   # 5% dữ liệu được coi là bất thường
  MIN_RECORDS = 30        # Cần ít nhất 30 bản ghi để train

Luồng xử lý (hàm detect_anomalies):
  1. Nhận list records từ DB
  2. Nếu len(records) < MIN_RECORDS → trả về rỗng
  3. Chuyển thành DataFrame
  4. Điền null bằng median của từng cột feature
  5. Chuẩn hóa với StandardScaler
  6. Fit IsolationForest (train trên TOÀN BỘ dữ liệu thành phố)
  7. Predict: -1 = bất thường, 1 = bình thường
  8. Tính anomaly_score = -decision_function (cao = bất thường hơn)
  9. Gán is_anomaly = True/False cho từng bản ghi
  10. Trả về list records đã annotate

Lưu ý kỹ thuật:
  - Model được re-train mỗi request (không lưu model)
  - Điểm yếu: chậm với dataset lớn
  - Cần review: xem tasks.md nhóm 1.1

IV. INFRASTRUCTURE & DOCKER
==============================================================

File: docker-compose.yml
  Services:
  1. redis
     - Image: redis:7-alpine
     - Port: 6379:6379
     - Volume: ./infra/redis/redis.conf
     - Healthcheck: redis-cli ping

  2. api
     - Build: services/api/
     - Port: 8000:8000
     - depends_on: redis
     - env_file: .env

  3. pipeline
     - Build: data-pipeline/
     - Không expose port (background service)
     - depends_on: redis
     - restart: unless-stopped
     - env_file: .env

  Lưu ý:
  - Supabase là cloud service — KHÔNG có trong Docker
  - Frontend (Next.js) chạy riêng ngoài Docker (npm run dev)

Lệnh thường dùng:
  docker compose up -d redis          # Chỉ khởi động Redis
  docker compose up -d                # Khởi động toàn bộ
  docker compose logs -f api          # Xem log API
  docker compose down                 # Dừng tất cả
