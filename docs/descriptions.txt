📄 TÀI LIỆU MÔ TẢ DỰ ÁN (PROJECT DESCRIPTION)
Tên đề tài: Weather Monitor - Hệ thống Giám sát và Phân tích Thời tiết Thông minh
Quy mô triển khai: Lãnh thổ Việt Nam
Nền tảng: Web Application (Interactive Dashboard)

1. TỔNG QUAN DỰ ÁN (PROJECT OVERVIEW)
1.1. Mục tiêu
Xây dựng một nền tảng Web Dashboard cung cấp thông tin thời tiết trực quan, tự động cập nhật (near-realtime). Hệ thống không chỉ dừng lại ở việc hiển thị số liệu mà còn tích hợp các kỹ thuật Phân tích Dữ liệu (Data Analysis) và Trí tuệ Nhân tạo (AI) để phát hiện bất thường, cảnh báo sớm và tự động tóm tắt tình hình thời tiết, giúp người dùng ra quyết định nhanh chóng.

1.2. Nguồn dữ liệu (Data Sources)

WAQI (World Air Quality Index): Thu thập dữ liệu chất lượng không khí (AQI).

Weather API: Thu thập thông tin khí tượng (nhiệt độ, độ ẩm, sức gió, lượng mưa, UV...).

2. KIẾN TRÚC HỆ THỐNG & CÔNG NGHỆ (TECH STACK)
Hệ thống được thiết kế theo kiến trúc module hóa, sử dụng các công nghệ hiện đại và tối ưu cho dữ liệu chuỗi thời gian (time-series):

Data Pipeline: Python (Requests, Pandas), Cron job/Airflow để fetch và clean dữ liệu định kỳ.

Database & Backend Service: * Supabase (PostgreSQL): Lưu trữ dữ liệu hệ thống, tự động sinh REST API cho Frontend, quản lý xác thực.

Redis: Bộ nhớ đệm (Cache) giúp giảm tải Database và tăng tốc độ phản hồi cho Dashboard.

Python (FastAPI): (Tùy chọn) Chạy như một microservice riêng để xử lý các logic tính toán AI/ML nặng.

AI & Data Analysis: Python (Scikit-learn, Matplotlib, Seaborn cho khâu phân tích EDA và huấn luyện model anomaly detection), LLM API (Gemini/GPT cho phần tóm tắt văn bản).

Frontend: ReactJS / Next.js, Tailwind CSS (giao diện), React-Leaflet / Mapbox (bản đồ), Recharts / Chart.js (biểu đồ).

Môi trường & Công cụ: Git/GitHub, Docker, Visual Studio Code.

3. CÁC TÍNH NĂNG CỐT LÕI (CORE FEATURES)
Bản đồ Thời tiết Tương tác (Interactive Weather Map): Hiển thị bản đồ Việt Nam với các marker/heatmap thể hiện nhiệt độ, AQI theo từng trạm đo.

Dashboard Phân tích Chuyên sâu: Biểu đồ xu hướng (trend) của nhiệt độ, độ ẩm, lượng mưa trong 24h qua và dự báo 7 ngày tới.

Hệ thống Cảnh báo Bất thường (Anomaly Alerts): Tự động phát hiện và cảnh báo (UI badge/notification) khi có các chỉ số cực đoan (nắng nóng đột ngột, AQI nguy hại).

AI Weather Summary: Sinh văn bản tóm tắt tự động bằng AI (VD: "Hôm nay Hà Nội ô nhiễm không khí nặng, chỉ số tia UV cao vào buổi trưa, khuyến cáo hạn chế ra đường").

4. PHÂN CÔNG NHIỆM VỤ (TEAM ROLES & RESPONSIBILITIES)
(Dành cho Team 4 người)

👤 1. Data Engineer (Data Pipeline & Database)

Viết script Python để gọi API từ WAQI và WeatherAPI.

Làm sạch (Clean), xử lý dữ liệu bị thiếu (Missing data) và chuẩn hóa định dạng.

Thiết kế Schema cho bảng dữ liệu chuỗi thời gian trên PostgreSQL (Supabase).

Set up Cron job đẩy dữ liệu tự động, liên tục vào Database.

👤 2. Backend Engineer (APIs, Cache & System)

Quản lý hạ tầng Supabase (Thiết lập Row Level Security, viết Edge Functions nếu cần).

Thiết lập và tích hợp Redis để cache dữ liệu trả về, giải quyết bài toán Rate Limit.

Đóng gói (Dockerize) các service của hệ thống và quản lý quá trình deploy (Render/AWS).

👤 3. AI / Data Analyst (Logic, Anomaly & Summary)

Thực hiện EDA (Exploratory Data Analysis) để hiểu phân phối của dữ liệu thời tiết Việt Nam.

Xây dựng thuật toán / model để phát hiện sự kiện thời tiết bất thường (Anomaly Detection).

Viết logic prompt engineering kết nối với LLM để tạo ra các đoạn text "AI Summary" tự nhiên, chính xác.

👤 4. Frontend Developer (UI/UX & Visualization)

Xây dựng giao diện web chuẩn Responsive bằng React/Next.js.

Tích hợp bản đồ địa lý và biểu đồ trực quan (Line chart, Bar chart).

Gọi API từ Supabase để lấy dữ liệu realtime và hiển thị cảnh báo lên màn hình.
